PROBLEM:
  ❌ When student/admin logs out, page doesn't open properly
  ❌ Unclear if logout was successful
  ❌ Session might persist

SOLUTION:
  ✅ Custom logout view clears session completely
  ✅ Success message confirms logout
  ✅ Automatic redirect to fresh login page
  ✅ Cannot access dashboard after logout

════════════════════════════════════════════════════════════════════════════════

🔧 IMPLEMENTATION DETAILS
════════════════════════════════════════════════════════════════════════════════

1. CUSTOM LOGOUT VIEW (hostel/views.py):
   ┌─────────────────────────────────────────────────────────┐
   │ def custom_logout(request):                             │
   │     logout(request)  # ← Clears session completely      │
   │     messages.success(...)  # ← Shows message            │
   │     return redirect('login')  # ← Redirects to login    │
   └─────────────────────────────────────────────────────────┘

2. URL MAPPING (hostel/urls.py):
   path('logout/', views.custom_logout, name='logout')

3. SESSION SECURITY (smart_hostel/settings.py):
   • SESSION_ENGINE = 'django.contrib.sessions.backends.db'
   • SESSION_COOKIE_HTTPONLY = True
   • SESSION_COOKIE_SAMESITE = 'Lax'
